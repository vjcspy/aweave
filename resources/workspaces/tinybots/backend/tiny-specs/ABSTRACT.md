**tiny-specs** is the centralized OpenAPI specification repository for TinyBots backend services. It generates TypeScript types and runtime validators from OpenAPI schemas, ensuring type consistency between backend services and frontend applications. tiny-specs generates two separate bundles: To add schemas for a service (e.g., micro-manager v6): 1. **Create/update schemas:** specs/local/components/micro-manager/v6/schemas.yaml 2. **Create/update paths:** specs/local/paths/micro-manager/v6/paths.yaml 3. **Update main file:** specs/local/micro-manager-main.yaml (reference new paths) 4. **Build:** yarn run all When developing new schemas, use a feature branch: **IMPORTANT:** When updating tiny-specs, use the dedicated command: This command provides: Pre-flight checks (clean working directory, correct branch) Step-by-step workflow for schema changes Integration steps for dependent repositories Error handling guidance All datetime fields should be ISO8601 strings: Use $ref for reusable components: Reference common error schemas: **Source Code:** tinybots/backend/tiny-specs/ **AI Command:** agent/commands/tinybots/update-tiny-specs.md **Build Output:** tiny-specs/dist/
