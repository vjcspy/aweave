# Wonkers Taas Orders — Abstract

Node/Express service that handles the full TaaS order lifecycle (create, patch, status transitions, returns) for both external partners and dashboard admins. Provides multiple API versions (v3–v6) plus internal endpoints, with Kong-authenticated access control and DTO validation. Orchestrates MySQL persistence, TinyBots TaaS subscriptions, PostNL delivery checks, Monday TEx board creation, device status updates, and rich email notifications. Runs a cron-driven delivery checker and depends on config-driven SMTP, Kong, TaaS, Nedap Ons, Monday, and Device Signals services. Source of truth for creating, updating, and tracking TaaS hardware orders and related concept orders/returns. Acts as the glue between integration users (external), dashboard admins, and downstream services (TaaS service, device signals, email/Slack/Monday).
